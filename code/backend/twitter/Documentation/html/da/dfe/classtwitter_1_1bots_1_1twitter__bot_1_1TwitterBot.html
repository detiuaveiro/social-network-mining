private

Function to get an user's timeline tweets.

At the same time, this function send to the server requests to like a tweet, to retweet a tweet and also sends all users that made some tweet, but we don't know yet. Also, this function is recursive, and tries to get all this data to the new users it possibility find

Parameters
user(User) user object to read the timeline for
jump_users(bool) flag to know if we should jump between tweet's users or not
max_depth(int) maximum recursion's depth
current_depth(int) current recursion's depth (to control the depth of the function's recursion)

Function to get the 20 (default) most recent tweets (including retweets) from some user.

Parameters
user(User) user whom tweets we want
  • since_id: Returns only statuses with an ID greater than (that is, more recent than) the specified ID
  • max_id: Returns only statuses with an ID less than (that is, older than) or equal to the specified ID
  • count: Specifies the number of statuses to retrieve
  • page: Specifies the page of results to retrieve. Note: there are pagination limits
